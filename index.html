    Project Idea
    Troy’s I/O Engine (TI/OE):

    Overview:

    A free and Open Source program for Windows 10 computers that allows the user to easily combine multiple input devices into one Virtual controller. This virtual device can be used on Simulators and Games that do not allow the user to mix inputs from multiple devices natively. TI/OE will be streamlined as not to confuse non-technical Users with options they do not need. It will also allow Users to share their configurations with each other and most importantly, be low latency and provide great performance

    Motivation:

    I have movement difficulties, particularly on my right side, namely my right arm, elbow, hand, right leg, hip and foot. While I have close to full movement range, there is large amount of discomfort at times, which can make which can make using some Gaming devices quite a challenge. When trying to figure out a way I can continue to play games at a high level, I realised that instead of reinventing the wheel with custom made devices, I could just adapt the many, many devices I have to my needs.
    For example, I have trouble pressing the Right triggers on a Gamepad for an extended period of time, however my left foot is perfectly fine, so why cannot I use the pedals from my Racing Sim Wheel setup to emulate the right triggers? Alternatively, I find using my right foot hard in Racing Sims, so why cannot I use a Game pad in my left hand to simulate the Right Pedal?

    The only options right now are expensive hardware options, or closed ecosystem paid software. I want to enable all gamers to have fun, on their own terms, for free.

    Description:

    TI/OE is a Windows 10 application that will allow the user to combine multiple Input devices together into one virtual device. This virtual device will allow many unique configurations of devices to be used in applications which do not support them. At present, it is planned to support HID, DirectInput and XInput physical devices, and HID, DirectInput and XInput as virtual devices.
    The user will be able to select a Virtual Controller template from a list of commonly used options, such as Fight sticks, Game pads, Steering wheels, Pedals and Gear sticks, Flight sticks (also known as Yokes) and custom hybrid configurations.
    There will be options for users share their templates and configurations, via the built in TI/OE Configuration Tool, this will allow users to share, browse and download configurations using the TI/OE website as a repository. There will also be a console which will display relevant low level information, such as a verbose mode to assist in troubleshooting any issues that may arise. In addition, there will be a real time latency display for each device. This is so you can determine the latency of your USB devices, which is extremely desirable information for many gamers, and very hard information to find. This can also help a user troubleshoot the health of their USB devices.
    An additional feature is the ability to bind a keypress or device input to a keystroke. If you wish to use a gaming device with a program that does not accept input from DirectInput or XInput, this will circumvent this issue. What if you want to use a foot pedal to copy the highlighted text in your word processor? And another pedal to paste? You will be able to bind one to CTRL+C and the other to CTRL+V. I acknowledge the ability to bind key pressed to gamepads is feature is available in other programs, however, I feel that if you need this functionality, it should be included in the one “do it all” application as opposed to forcing the user to run another application in conjunction with TI/OE.
    When running TI/OE will minimize to the taskbar and have a very low overhead. The system requirements will be very low, as to enable anyone with a Windows 10 computer to use this application.
    Finally another feature I think will be extremely desirable, is the ability for TI/OE to report the battery level of your Bluetooth USB devices. Currently, when a Bluetooth Xbox or PS4 Gamepad is connected to a PC, you need to find a 3rd party application to report the battery level. I believe this feature along with the USB Latency meter will make TI/OE a very useful application, even if you have no desire to use the other features of TI/OE.

    Tools and Technologies

    The requirements to use TI/OE are a Windows 10 PC and some USB devices you with to use with TI/OE. However due to limitations of your PC, you may wish to use a USB hub if you intend to use more USB devices than the number of available USB ports. An Internet connection is required to visit the TI/OE website and download user shared configurations and content.

    Skills Required

    The skills required to complete this project are as follows. A working knowledge of Windows 10 and the XInput API. In addition, knowledge of the Legacy API DirectInput for older and not-Windows devices, such as the Sony DS3 Gamepad and many older Fight sticks. As I am currently learning Java, I would endeavour to complete the project in this language.
    To enable users to be able to share their configurations on a central server, a low spec virtual Web server should be ideal, HTML, XML, and SQL would be a great starting point as the configurations will be in XML and have very low bandwidth requirements. Many existing Content Management Systems software packages could be utiltized for this purpose.

    Outcome

    If this project is successful, it will enable more users to empower themselves to play games their own way, without paying large sums of money, or being stuck in a closed source ecosystem. I hope to one day implement TI/OE as a stand alone Raspberry Pi distribution, that can turn a Raspberry Pi into a I/O engine that will work on any console, any PC, configurable via a web interface on the Raspberry Pi. If someone finds this application and it enabled them to find happiness in video games, then I believe it will have been successful.
